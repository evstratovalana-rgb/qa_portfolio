# Тестирование сервиса «Яндекс Самокат»

## Цель проекта
Тестирование веб и мобильной версий сервиса аренды самокатов, включая проверку ключевых пользовательских сценариев и интеграцию между платформами.

## Роль
**QA Engineer** (индивидуальный проект)  
**Охват:** Веб-версия (формы заказа) + Мобильное приложение (регрессионное тестирование)

**Ключевые задачи:**
1.  Анализ требований и разработка тестовой документации для веб-форм.
2.  Тестирование веб-версии с применением техник тест-дизайна.
3.  Регрессионное тестирование мобильного приложения по готовым тест-кейсам.
4.  Работа с API для подготовки тестовых данных.
5.  Анализ результатов и формирование итогового заключения.

## Работа и артефакты

### Тест-анализ и дизайн (Веб-версия)
*   **Анализ требований:** Изучила спецификации ключевых форм («Для кого самокат», «Про аренду»)
*   **Чек-лист:** Разработала структурированный чек-лист для проверки веб-форм
    *   [Пример чек-листа](checklist_example.md)
*   **Техники тест-дизайна:** Применила на практике:
    - Классы эквивалентности
    - Граничные значения
    - Попарное тестирование (Pairwise)

### Тестирование веб-версии
*   **Кросс-браузерное тестирование:** Провела проверки в Яндекс.Браузере и Google Chrome
*   **Функциональное тестирование:** Полный цикл проверки форм заказа самоката
*   **Баг-репортинг:** Выявила, воспроизвела и детально оформила найденные дефекты
    *   [Примеры баг-репортов](bug_reports_examples.md)

### Регрессионное тестирование мобильного приложения
*   **Методология:** Выполнение регрессионных проверок по готовым тест-кейсам для контроля стабильности
*   **Цель:** Убедиться, что новые изменения не сломали существующую функциональность
*   **Окружения:**
    - **Реальное устройство:** Тестирование на физическом Android-смартфоне
    - **Эмулятор:** Использование Android Studio для быстрых проверок
*   **Ретестинг дефектов:** Проверка исправленных багов на реальном устройстве

### Работа с API (Postman)
*   **Инструмент:** Postman для работы с REST API сервиса
*   **Цель:** Создание тестовых заказов для формирования необходимых данных
*   **Реализация:**
    - Создание коллекции запросов для API Яндекс.Самокат
    - Настройка тестовых окружений и переменных
    - Генерация данных для комплексной проверки связанного функционала

### Итоговый анализ
*   **Сводка результатов:** Проанализировала итоги тестирования по всем направлениям
*   **Оценка готовности:** Сформировала вывод о готовности функциональности к релизу
    *   [Итоговый отчёт](reports_summary.md)

 **Кросс-платформенное тестирование** — веб + мобильное приложение + API  
 **Применение техник тест-дизайна** на практике
