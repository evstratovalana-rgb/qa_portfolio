# Примеры баг-репортов

## 1: Критичный баг — не работает оплата через СБЕР

**ID:** POMOGI-63  
**Заголовок:** Не работает кнопка "пожертвовать" через форму пожертвования "Онлайн"-СБЕР  
**Приоритет:** Высокий  
**Статус:** Открыт  
**Автор:** Светлана Евстратова

### Описание
При попытке совершить пожертвование через СБЕР с указанием предложенной суммы (250₽, 500₽ и т.д.) платеж не проходит. Кнопка "пожертвовать" меняет состояние на "отправляем" и возвращается обратно.

### Шаги воспроизведения
1. Открыть страницу семьи: `https://pl.rassokha.pro:3001/family_request/18`
2. В блоке "Помочь ребенку!" выбрать способ помощи: "СБЕР"
3. Выбрать предложенную сумму (например, 250 рублей)
4. Заполнить обязательные поля:
   - Имя: "Ми"
   - Email: `test@yandex.ru`
5. Поставить галочку согласия с условиями
6. Нажать кнопку "Пожертвовать"

### Ожидаемый результат
Система перенаправляет на страницу оплаты СБЕР, платеж инициирован.

### Фактический результат
Кнопка "Пожертвовать" на секунду меняет текст на "отправляем", затем возвращается в исходное состояние. Перехода к оплате не происходит.
**Окружение:** Xiaomi HyperOS 2.0.207.0, Android 15, Google Chrome

## 2: Проблема с валидацией формы волонтера

**ID:** POMOGI-31  
**Заголовок:** Не отображается красный текст ошибки в поле Email при клике по полю и дальнейшему переходу к следующему полю  
**Приоритет:** Высокий  
**Статус:** Открыт  
**Автор:** Светлана Евстратова

### Описание
В форме "Анкета волонтера" при оставлении обязательного поля Email пустым система не показывает ошибку валидации. Пользователь не понимает, почему кнопка отправки неактивна.

### Шаги воспроизведения
1. Перейти на страницу: `https://pl.rassokha.pro:3001/volunteer-applications`
2. Заполнить все поля кроме Email:
   - Фамилия: "Васильев"
   - Имя: "Василий"
   - Отчество: "Васильевич"
   - Телефон: `+79998887744`
   - Город: "Москва"
   - Дата рождения: "1992-09-09"
   - Фото: выбрать файл
   - Род занятий: "Врач"
3. Поле Email оставить пустым
4. Кликнуть на следующее поле (например, "Город")
5. Поставить галочку согласия

### Ожидаемый результат
- Кнопка "СТАТЬ ВОЛОНТЕРОМ" неактивна
- Под полем Email красный текст: "поле обязательно для заполнения"

### Фактический результат
- Кнопка неактивна
- Ошибка валидации не отображается
- Пользователь не получает обратной связи о проблеме
**Окружение:** Xiaomi HyperOS 2.0.207.0, Android 15, Google Chrome

## Как я оформляю баг-репорты

### Стандартная структура:
1. **ID и заголовок** — уникальный номер + что не так
2. **Приоритет** — High/Medium/Low на основе влияния
3. **Шаги воспроизведения** — четко, чтобы любой мог повторить
4. **Ожидаемый vs Фактический** — контраст "как должно быть" и "как есть"
5. **Окружение** — где проверял (устройства, браузеры)
6. **Доказательства** — скриншоты, ссылки на макеты/требования

### Мой подход к приоритизации:
- **High:** Блокирует основную функциональность (платежи, формы)
- **Medium:** Ухудшает UX, но можно использовать
- **Low:** Косметические проблемы, опечатки
