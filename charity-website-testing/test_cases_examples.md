# Примеры тест-кейсов

**Ключевые направления:** Функциональность, Производительность, Надежность, Usability

## Тест-кейсы для мобильных устройств (Web-mobile)

*Особенности: Touch-интерфейс, адаптивная верстка, смена ориентации*

### TC-09: Сохранение данных формы при смене ориентации
**Приоритет:** Высокий  
**Что проверяет:** Корректность работы state management на мобильных

**Шаги:**
1. На мобильном устройстве заполнить форму донации (сумма, email, галочка согласия)
2. Повернуть устройство из вертикальной в горизонтальную ориентацию

**Ожидаемый результат:** Все введенные данные сохраняются

---

### TC-11: Устойчивость к обрывам соединения
**Приоритет:** Высокий  
**Что проверяет:** Отказоустойчивость формы при проблемах с интернетом

**Шаги:**
1. Заполнить форму донации на мобильном устройстве
2. Отключить Wi-Fi и мобильный интернет
3. Подождать 10 секунд
4. Включить интернет обратно

**Ожидаемый результат:** Данные не потеряны, форма в прежнем состоянии

---

## Тест-кейсы для десктопа (Web)

*Особенности: DevTools, производительность, hover-эффекты*

### TC-05: Время загрузки при медленном интернете
**Приоритет:** Высокий  
**Инструмент:** Chrome DevTools → Network → Throttling

**Шаги:**
1. Открыть DevTools → вкладка Network
2. Установить ограничение скорости: 10 Mbps
3. Обновить страницу сайта
4. Зафиксировать время загрузки (параметр "Finish")

**Ожидаемый результат:** ≤ 3 секунд  

---

### TC-13: Работа модальных окон (hover и клики)
**Приоритет:** Средний  
**Что проверяет:** Usability интерфейса на десктопе

**Шаги:**
1. Навести курсор на название видео в разделе "Вы помогли этим детям!"
2. Проверить hover-эффект (подчеркивание)
3. Кликнуть на видео

**Ожидаемый результат:**
1. При наведении появляется подчеркивание
2. По клику происходит переход на видео в ВК

---

## Тест-кейсы для всех платформ (All)

*Особенности: Кросс-платформенная функциональность, интеграции*

### TC-12: Интеграция с платежной системой ЮKassa
**Приоритет:** Высокий (критичный функционал)  
**Что проверяет:** Полный цикл донации от формы до бэкенда

**Шаги:**
1. Заполнить форму донации (имя: "Юрий Долгорукий", сумма: 250₽, email)
2. Принять условия оферты
3. Нажать "ПОМОЧЬ С КАРТЫ"
4. В окне ЮKassa ввести тестовые данные карты
5. Подтвердить платеж
6. Проверить логи бэкенда: `https://pl.rassokha.pro:8888/container/...`

**Ожидаемый результат:**
```json
metadata: {
  first_name: 'Юрий Долгорукий',
  email: 'testforms89@mail.com',
  agreement: 'true'
}
```

### TC-01: Навигация по логотипу
**Приоритет:** Средний  
**Что проверяет:** Базовую навигацию и узнаваемость бренда

**Шаги:**
1. Перейти на страницу "Кому помогли": `https://pl.rassokha.pro:3001/whom-we-helped`
2. Кликнуть на логотип в левом верхнем углу

**Ожидаемый результат:**
1. Логотип соответствует макету
2. Происходит переход на главную страницу
