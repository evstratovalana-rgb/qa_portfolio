# Тестирование сайта благотворительного фонда

## Цель проекта
Провести комплексное тестирование веб-сайта в составе большой команды: оценить функциональность, удобство использования, производительность и безопасность.

## Роль и команда
**Роль:** QA Engineer в команде из 8 тестировщиков  
**Команда:** Product Manager, Team Lead, 8 QA Engineers  

**Ключевые задачи:**
1.  Анализ требований к интерфейсу (главная страница, хедер, футер, боковое меню).
2.  Тест-дизайн и создание тестовой документации для своей зоны ответственности.
3.  Функциональное тестирование и баг-репортинг.
4.  Тестирование производительности и безопасности (HTTPS) в рамках своей части.
5.  Участие в командных обсуждениях и формулировка рекомендаций по улучшению.
6. Изучение документации API и тестирование эндпоинтов

## Артефакты и процесс

### Планирование и дизайн тестов
*   **Анализ требований:** Выявлено 4 «серые зоны» для уточнения.
*   **Чек-лист:** Разработан **чек-лист** для своей зоны ответственности, согласованный с Team Lead.
    *   [Пример чек-листа](checklist_example.md)
*   **Тест-кейсы:** Составлены **тест-кейсы для основных пользовательских сценариев** в рамках назначенных модулей.
    *   [Примеры тест-кейсов](test_cases_examples.md)

### Исполнение тестов и отчётность
*   **Баг-репорты:** Выявлены и оформлены дефекты.
    *   [Примеры оформленных баг-репортов](bug_report_examples.md)
*   **Тестирование API:** 
    *   [Скриншот коллекции запросов](postman_api.png)
*   **Специализированное тестирование:** Проведено тестирование производительности и безопасности для своей части сайта.
*   **Взаимодействие в команде:** Консультации с коллегами по сложным тестовым сценариям и взаимная проверка тест-кейсов и чек-листов.

### Итоги и рекомендации
По итогам тестирования участвовала в формировании общих рекомендаций команды.

## Используемые технологии и инструменты
`Web Testing` `Chrome DevTools` `Postman` `REST API` `Техники тест-дизайна` `Баг-репортинг` `Работа в большой команде` `Координация с PM и Team Lead`
