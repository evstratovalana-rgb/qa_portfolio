# Тестирование веб-приложения Stellar Burgers

## Цель проекта
Провести тестирование ключевых модулей веб-приложения в составе небольшой команды тестировщиков.

## Роль и команда
**Роль:** QA Engineer в команде из 5 тестировщиков  
**Команда:** 5 QA Engineers (без выделенного PM/Team Lead)  
**Мой вклад:** Ответственность за модуль аутентификации и ленту заказов

**Ключевые задачи:**
1.  Самостоятельный анализ требований и тест-дизайн для назначенных модулей.
2.  Создание полного набора тестовой документации для своих компонентов.
3.  Функциональное тестирование и оформление баг-репортов по выявленным проблемам.
4.  Активное участие в самоорганизации команды и распределении задач.
5.  Подготовка итогового отчета ппо результатам всей команды.

## Артефакты и процесс

### Планирование и дизайн тестов
*   **Анализ требований:** Провела для модулей аутентификации и ленты заказов
*   **Чек-листы:** Разработала для **UI/UX тестирования** закрепленных модулей
    *   [Примеры из чек-листа](checklist_examples.md)
*   **Тест-кейсы:** Создала с использованием техник тест-дизайна для ключевых сценариев
    *   [Примеры тест-кейсов](test_cases_examples.md)
- **Визуализация тестового покрытия:** Разработана майнд-карта архитектуры приложения
  - [Подробное описание и файлы майнд-карты](mindmaps/README.md)
        
### Исполнение тестов и отчётность
*   **Баг-репорты:** Выявлены и детально оформлены дефекты, влияющие на пользовательский опыт.
    *   [Примеры оформленных баг-репортов](bug_reports_examples.md)
*   **Командная самоорганизация:** Обсуждение и совместное принятие решений с 4 другими тестировщиками.

### Итоги и рекомендации
**Подготовила итоговый отчет по результатам работы всей команды** и на его основе сформировала вывод о готовности продукта к релизу.
*   [Пример командного отчёта](reports_team_summary.md)
