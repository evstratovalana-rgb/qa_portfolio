# Чек-лист для тестирования модуля аутентификации и ленты заказов

**Проект:** Stellar Burgers  
**Модуль:** Аутентификация (логин, восстановление пароля) и лента заказов  
**Моя зона ответственности:** Только авторизация (вход, восстановление пароля) и лента заказов  
*Примечание: Регистрация (создание аккаунта) тестировалась другим членом команды*

---

## **Цель моего тестирования**
Проверить сценарии **входа в существующий аккаунт** и работу **публичной ленты заказов**.

---

## **Аутентификация: Вход в аккаунт** (самостоятельно разработанные проверки)

### Страница авторизации (логин)
| ID | Проверка | Обоснование |
|----|----------|-------------|
| 1 | При клике на «Личный кабинет» открывается страница входа | Проверка основного пути доступа |

| ID | Проверка | Обоснование |
|----|----------|-------------|

| 6 | При валидном email — лоадер и переход на сброс пароля | Позитивный сценарий |
| 7 | Кнопка «Войти» возвращает на страницу авторизации | Навигация |

### Сброс пароля
| ID | Проверка | Обоснование |
|----|----------|-------------|

## **Мой подход к тест-дизайну для моего модуля**

### 1. **Чёткое понимание scope:**
-  **Моя зона:** Вход (логин) для существующих пользователей + публичная лента
-  **Не моя зона:** Регистрация новых пользователей

### 2. **Фокус на пользовательских сценариях:**
- **Существующий пользователь** забыл пароль → восстановление → вход
- **Любой посетитель** хочет посмотреть текущие заказы → лента заказов

### 3. **Комбинация проверок для входа:**
| Тип проверки | Пример | Цель |
|--------------|--------|------|
| Позитивная | Корректный логин/пароль | Основной сценарий |
| Негативная | Неверный пароль | Обработка ошибок |
| Альтернативная | Восстановление пароля | Резервный путь |
| Безопасность | Скрытие пароля | Защита данных |

### 4. **Особое внимание ленте заказов:**
- **Публичный доступ** (не требует авторизации)
- **Real-time обновления** (техническая сложность)
- **Удобство просмотра** (адаптивность, модальные окна)
