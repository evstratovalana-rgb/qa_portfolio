## Методология создания тест-кейсов

### Подход на основе таблицы принятия решений:
Для формы входа использовалась матрица 3×3:
- **Email:** {валидный, невалидный, пустой}
- **Пароль:** {валидный, невалидный, пустой}

Это позволило создать **9 тест-кейсов**, полностью покрывающих все возможные комбинации.

### Для форм восстановления пароля:
- Разделение на успешные и неуспешные сценарии
- Учет разных состояний данных (валидные/невалидные/пустые)
- Проверка переходов между страницами
